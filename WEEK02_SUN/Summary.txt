		: 명령어
			cd /etc/apt 
				: 솦웨어 설치와 관련된 파일이 저장된 디렉토리로 이동
				ls 해봄 sources.list 파일이 있다

			 sudo mv sources.list sources.list.back
				: 현재 게스트 os 로 로그인 되어있는 상태니까 sudo 를 통해 잠시 관리자 권한을 빌려온다는 의미로 sudo
				: 기존 sources.list 를 삭제하되 혹시라도 다시 필요할 수 있으니 백업용으로 다른 이름으로 저장해둔다
	
			sudo wget http://dw.hanbit.co.kr/ubuntu/20.04/sources.list 
				: 실습에 필요한 버전의 sources.list 를 다운받는다

			sudo apt update
				: advanced package tool 을 업데이트 시킨다
			

	: 게스트 pc에 IP 주소를 설정하기
		: 명령어
			ip addr
				: 나온 문구 중에서 ens32 부분의 ip 를 확인한다
					: 앞서 VMware pro에서 3번째 자리를 111 로 고정했으니 192.168.111.XXX로 주소가 나온다
				: ip a 명령어로도 가능하고, 이때는 현제 인터페이스(장치) 관련 ip 정보를 확인 가능하다
			
			cd /etc/netplan
				: netplan == 네트워크 인터페이스 대한 정적 ip 주소를 설정하는데 사용되는 파일
				: ls 해보면 확장자가 yaml 인 파일이 있다
					: netplan 설정파일의 확장자가 yaml 이다

			sudo nano 00-installer-config.yaml
				:  00-installer-config.yaml 파일을 편집한다
				: 수정 내용
					: 기존 00-installer-config.yaml 파일 내용
						#This is the network config written by 'subsiquity'
						network:
					           ethernets:
						      ens32:
							 dhcp4: true
                                                     version: 2
						
							: 내용 해석
								: network == Netplan 설정의 시작을 나타냄
								: version == Netplan 설정의 버전 지정
								: ethernets == 네트워크 인터페이스 대한 설정을 시작
								: ens32 == 네트워크 인터페이스 중 하나
								: "네트워크" "인터페이스"인 "ens32"의 "dhcp4"를 true 로 설정되어있어 자동으로 IP 주소를 할당 받는단 뜻이고, 별도의 설정 없이도 네트워크가 작동된다.  

					: 바꾼 내용
						#This is the network config written by 'subsiquity'
						network:
					           ethernets:
						      ens32:
							 dhcp4: no
							 addresses: [192.168.111.200/24]
						         gateway4: 192.168.111.2
                                                         nameservers:
								addresses: [192.168.111.2]
                                                     version: 2

								: 주석문 바로 밑 부분에 아래 내용을 입력하는데 띄어쓰기 틀리면 좆되는거다
							: 내용 해석
								: addresses == 인터페이스에 할당할 정적 ip 주소를 나타냄
								: [192.168.111.200/24] 에서 /24 == 서브넷 마스크
								: gateway4 == 게이트웨이(시스템이 외부와 통신하기 위한 경로)의 IP 주소 지정하는거
								: nameservers == dns 서버에 대한 설정
								
			ip addr
				: ip 제대로 바뀌었는지 확인한다

			ping -c 10 www.google.com
				: 네트워크 연결이 잘 됬는지 확인. google 로 ping을 10번 보내겠단 뜻
				: ping == 네트워크 상태를 확인하기 위해 사용되는 명령어로, 지정된 호스트(사이트주소) 에 ICMP 패킷을 보내고 응답 시간을 측정	
					: ICMP 
						: internet control message protocol
						: 네트워크에서 메시지를 교환하는데 사용되는 프로토콜.규약.
 
				

	: 관리자 root 의 비번을 지정하기

			sudo su - root

			 passwd
				: 관리자의 비밀번호를 지정한다
				
			reboot
				: 다 했음 reboot 한다

		: 관리자의 비번까지 생성했으면 관리자 계정ㅇ까지 만들어진거라 이제 초기에 로그인 할때 관리자로 로그인 가능하다( 처음에 root 입력하고(사용자명), 설정한 관리자 비번으로 로그인)
