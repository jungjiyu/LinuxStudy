리눅스 기본 명령어
	ls 
		: list 의 약자
		: windows 의 dir 명령과 같은 역할을 함
			: 디렉토리에 있는 파일들의 목로글 나열해즘

		: 옵션 >> -al 을 젤 많이 쓰긴함
			-a : 숨김파일까지 보여줌
			- l : 리스트의 형태로 세부정부까지 보여줌

		: ls -al  /home/
			하면 사용자들 모두 뜬다
				: 왜냐면 home 디렉터리에 모든 곚정자의 폴더가 저장되니까

		: 뒤에 경로 쓰면 현재 디렉토리 뿐 아니라 그 경로에 해당하는 디렉토리의 파일 목록까지 볼 수 있다.
	| : 파이프 라인
	cd 
		: change directory 의 약자
		: 옵션
			: 걍 cd >> 홈디렉터리 로 이동
			: cd ~계정자명 >> 해당 계정자의 홈 디렉터리로 이동
		: cd .. , cd . 이런거 다 cd 랑 붙여서 쓰는게 아니라 한칸 띄우고 하는거다
		: 경로 젤 앞에 / 있으면 루트 디렉토리 의 하위 그런거고 젤 앞에 / 없으면 현재위치 기준 하위 그런거 의미하는거다
		: 경로 젤 뒤에 / 있으면 디렉토리 경로로 끝나는 거고 끝에 / 없으면 파일을 말하는거 같다

	pwd
		: print working directory
		: 현재 디렉토리의 전체 경로를 보여줌

	rm
		: 파일을 삭제
		: 옵션
			: -i ) 삭제시 정ㅁ말 삭제할지 확인하는 메셎 ㅣ출력
				: 대답은 y, n 로 함
			: -f ) 삭제시 확인 없이 삭제
				: force
			: -r ) 폴더를 삭제
				: recursive ==하위에 있는 얘들까지
			: -rf ) 강제로 전부 삭제
		: root 관리자거나(모든 권한이 있으니까) 아니면 사용자가 삭제할 권리를 가지고 있어야됨
		: 원본파일을 삭제하기 이전에 백업파일용 만들어놓고 삭제하길 추천한다
			: cp 원래파일명 원래파일명.back 로 저장해둔 후 rm -rf 한다
				: 물론 이 백업파일을 다시 쓰는 경우 cp 원래파일명.back 원래파일명 으로 하면 된다

cp
	: copy 의 약자로 파일을 복사
	: 새로 복사한 파일은 복사한 사용자의 소유가 된다
		: 명령을 실행하는 사용자는 해당 파일의 읽기 권한이 필요함
	: 옵션 
		-r ) 디렉토릴 복사
	: 신기한 예시	
		cp folder1/a* folder2
			: folder2 에 folder1 디렉토리 자체가 아니라 a 로 시작하는 파일들만 folder2 로 복사한다
		cp 파일명 경로/새로운파일명
			: 해당 경로에 새로운 이름으로 복사한다

		cp -r folder1/* folder2
			: folder1 자체가 복사되는건 아니지만 folder1 내부의 모든 파일 , 폴더들이 folder2 로 복사된다.
